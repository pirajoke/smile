S.M.I.L.E. (AI Install) — Privacy Policy

Last updated: April 6, 2026

Overview

S.M.I.L.E. ("AI Install") is a browser extension that adds an "Install with AI" button to GitHub, GitLab, and Bitbucket repository pages. We are committed to protecting your privacy.

Data Collection

We do not collect, transmit, or store any personal data.

Specifically, the extension:

  • Does not track your browsing history
  • Does not send any data to external servers
  • Does not use analytics, telemetry, or tracking scripts
  • Does not read cookies or authentication tokens
  • Does not access any data outside of supported repository pages

Local Storage

The extension stores the following data locally on your device using Chrome's built-in storage API (chrome.storage.sync):

  • User preferences: default install tool, emoji pack selection, custom command template
  • NFT share history: last 10 emoji shares (emoji, public repo path, tier). This data contains only publicly available repository paths, not private information.

This data syncs across your Chrome browsers via your Google account but is never sent to any third party.

Permissions

  • clipboardWrite: To copy install commands to your clipboard
  • storage: To save your preferences locally
  • nativeMessaging: To optionally send commands to your local terminal (requires separate opt-in installation)

Third-Party Services

The extension does not integrate with any third-party services. Share buttons (Telegram, WhatsApp, X) open standard share URLs in a new tab — no data is sent until you explicitly confirm in the respective service.

Native Messaging Host

The optional native messaging component runs locally on your machine and executes terminal commands. It only accepts commands matching a strict allowlist (git clone, claude, codex, open). It does not communicate with any external server.

Open Source

This extension is fully open source. You can review the complete source code at github.com/pirajoke/smile.

Contact

If you have questions about this privacy policy, please open an issue at github.com/pirajoke/smile/issues.
